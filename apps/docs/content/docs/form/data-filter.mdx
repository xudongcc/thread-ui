```bash
pnpm dlx shadcn@latest add @thread-ui/data-filter
```

## Usage

```tsx
import {
  DataFilter,
  zhCN,
  type DataFilterItemProps,
  type DataFilterValue,
} from "@/components/thread-ui/data-filter";
```

```tsx
const filters: DataFilterItemProps[] = [
  {
    field: "name",
    label: "Name",
    type: "input",
    defaultOperator: "$fulltext",
    operators: ["$eq", "$ne", "$fulltext"],
    placeholder: "Filter by name...",
  },
  {
    field: "amount",
    label: "Amount",
    type: "number-input",
    defaultOperator: "$gte",
    operators: ["$eq", "$ne", "$gt", "$gte", "$lt", "$lte", "$between"],
    min: 0,
    max: 10000,
    decimalScale: 2,
  },
  {
    field: "age",
    label: "Age",
    type: "number-input",
    defaultOperator: "$between",
    operators: ["$eq", "$ne", "$gt", "$gte", "$lt", "$lte", "$between"],
    min: 0,
    max: 120,
  },
  {
    field: "createdAt",
    label: "Created At",
    type: "date-picker",
    defaultOperator: "$between",
    operators: ["$eq", "$ne", "$gt", "$gte", "$lt", "$lte", "$between"],
    min: new Date("2024-01-01"),
    max: new Date("2026-12-31"),
  },
  {
    field: "archived",
    label: "Archived",
    type: "checkbox",
    defaultOperator: "$eq",
    operators: ["$eq", "$ne"],
  },
];

const value: DataFilterValue = {
  query: "Acme",
  orderBy: {
    field: "createdAt",
    direction: "DESC",
  },
  filter: {
    amount: {
      $gte: 100,
    },
    age: {
      $between: [18, 30],
    },
    createdAt: {
      $between: ["2025-03-01T00:00:00.000Z", "2025-03-15T00:00:00.000Z"],
    },
    name: {
      $fulltext: "Acme",
    },
    archived: {
      $eq: true,
    },
  },
};

<DataFilter filters={filters} value={value} onChange={(v) => console.log(v)} />;
```

## Localization

`DataFilter` supports a DayPicker-style `locale` prop. Pass a locale code to use a built-in preset, or import a preset from the component locale exports.

```tsx
import { DataFilter, zhCN } from "@/components/thread-ui/data-filter";

<DataFilter filters={filters} locale="zh-CN" />;
<DataFilter filters={filters} locale={zhCN} />;
```

The built-in date picker receives the preset calendar locale automatically. You can also provide partial overrides, which are merged with the default English locale.

```tsx
<DataFilter
  filters={filters}
  locale={{
    addFilter: "Add condition",
    operators: {
      $fulltext: "matches",
    },
  }}
/>
```

### Async Select Options

Select filters can load options from an async function. The function receives the current query and returns matching options. When stored values may not appear in the current result page, provide `resolveSelectedOptions` to fetch selected option labels by value.

<Preview path="data-filter-async-select" />

### Custom Render Components

You can use built-in `input`, `number-input`, `date-picker`, and `checkbox` filters, or replace them with sophisticated components such as a `Select` drop-down or a `DatePicker` block via the `render` prop function. Custom renderers receive the selected `operator` and the raw field value. Use `renderValue` to specify how the formatted value appears in the badge.

### DataFilterOperator

Built-in filters support `$eq`, `$ne`, `$gt`, `$gte`, `$lt`, `$lte`, `$between`, `$fulltext`, `$in`, and `$nin`. Input filters support `$eq`, `$ne`, and `$fulltext`; number-input and date-picker filters support `$eq`, `$ne`, `$gt`, `$gte`, `$lt`, `$lte`, and `$between`; checkbox filters support `$eq` and `$ne`; select filters support `$in` and `$nin`.

The number-input and date-picker `$between` operators output a native range tuple inside `value.filter`, for example `{ age: { $between: [18, 30] } }` or `{ createdAt: { $between: ["2025-03-01T00:00:00.000Z", "2025-03-15T00:00:00.000Z"] } }`. A `$between` filter is included in `value.filter` only after both range endpoints are set.

All built-in filter types also support `is empty` and `is not empty`. These options output `$eq: null` and `$ne: null` respectively, and the value editor content is hidden while either option is selected.
