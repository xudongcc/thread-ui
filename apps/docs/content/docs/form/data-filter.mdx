```bash
pnpm dlx shadcn@latest add @thread-ui/data-filter
```

## Usage

```tsx
import {
  DataFilter,
  type DataFilterItemProps,
  type DataFilterValues,
} from "@/components/thread-ui/data-filter";
```

```tsx
const filters: DataFilterItemProps[] = [
  {
    field: "name",
    label: "Name",
    type: "input",
    defaultOperator: "$fulltext",
    operators: ["$eq", "$ne", "$fulltext"],
    placeholder: "Filter by name...",
  },
  {
    field: "amount",
    label: "Amount",
    type: "number-input",
    defaultOperator: "$gte",
    operators: ["$eq", "$ne", "$gt", "$gte", "$lt", "$lte", "$between"],
    min: 0,
    max: 10000,
    decimalScale: 2,
  },
  {
    field: "age",
    label: "Age",
    type: "number-input",
    defaultOperator: "$between",
    operators: ["$eq", "$ne", "$gt", "$gte", "$lt", "$lte", "$between"],
    min: 0,
    max: 120,
  },
  {
    field: "createdAt",
    label: "Created At",
    type: "date-picker",
    defaultOperator: "$between",
    operators: ["$eq", "$ne", "$gt", "$gte", "$lt", "$lte", "$between"],
    min: new Date("2024-01-01"),
    max: new Date("2026-12-31"),
  },
  {
    field: "archived",
    label: "Archived",
    type: "checkbox",
    defaultOperator: "$eq",
    operators: ["$eq", "$ne"],
  },
];

const values: DataFilterValues = {
  amount: {
    $gte: 100,
  },
  age: {
    $gte: 18,
    $lte: 30,
  },
  createdAt: {
    $gte: "2025-03-01T00:00:00.000Z",
    $lte: "2025-03-15T00:00:00.000Z",
  },
  name: {
    $fulltext: "Acme",
  },
  archived: {
    $eq: true,
  },
};

<DataFilter
  filters={filters}
  values={values}
  onChange={(v) => console.log(v)}
/>;
```

## Examples

### Standalone Search

You can use the search feature entirely on its own by providing an empty `filters` array.

<Preview path="data-filter-search" />

### Sort Menu

The Search bar provides an anchor for the `sort` control menu, allowing users to select directional grouping for different fields directly from the primary bar.

<Preview path="data-filter-sort" />

### Async Select Options

Select filters can load options from an async function. The function receives the current query and returns matching options.

<Preview path="data-filter-async-select" />

### Custom Render Components

You can use built-in `input`, `number-input`, `date-picker`, and `checkbox` filters, or replace them with sophisticated components such as a `Select` drop-down or a `DatePicker` block via the `render` prop function. Custom renderers receive the selected `operator` and the raw field value. Use `renderValue` to specify how the formatted value appears in the badge.

### DataFilterOperator

Built-in filters support `$eq`, `$ne`, `$gt`, `$gte`, `$lt`, `$lte`, `$between`, `$fulltext`, `$in`, and `$nin`. Input filters support `$eq`, `$ne`, and `$fulltext`; number-input and date-picker filters support `$eq`, `$ne`, `$gt`, `$gte`, `$lt`, `$lte`, and `$between`; checkbox filters support `$eq` and `$ne`; select filters support `$in` and `$nin`.

The number-input and date-picker `$between` operators output a range using `$gte` and `$lte`, for example `{ age: { $gte: 18, $lte: 30 } }` or `{ createdAt: { $gte: "2025-03-01T00:00:00.000Z", $lte: "2025-03-15T00:00:00.000Z" } }`.

All built-in filter types also support `is empty` and `is not empty`. These options output `$eq: null` and `$ne: null` respectively, and the value editor content is hidden while either option is selected.
