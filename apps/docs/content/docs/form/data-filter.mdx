## Installation

```bash
pnpm dlx shadcn@latest add @thread-ui/data-filter
```

## Usage

```tsx
import {
  DataFilter,
  type DataFilterItemProps,
  type DataFilterValues,
} from "@/components/thread-ui/data-filter";
```

```tsx
import { Input } from "@/components/ui/input";

const filters: DataFilterItemProps[] = [
  {
    field: "name",
    label: "Name",
    render: ({ field: { value, onChange } }) => (
      <div className="w-48 p-2">
        <Input
          value={value || ""}
          onChange={(e) => onChange(e.target.value)}
          placeholder="Filter by name..."
        />
      </div>
    ),
  },
];

const values: DataFilterValues = {};

<DataFilter
  filters={filters}
  values={values}
  onChange={(v) => console.log(v)}
/>;
```

## Examples

### Standalone Search

You can use the search feature entirely on its own by providing an empty `filters` array. You can also customize the search bar with `leading` and `trailing` elements.

<Preview path="data-filter-search" />

### Sort Menu

The Search bar provides an anchor for the `sort` control menu, allowing users to select directional grouping for different fields directly from the primary bar.

<Preview path="data-filter-sort" />

### Pinned Filters

Set `pinned: true` to make specific filters always visible and preventing the user from completely removing them. Provide `search: false` if you want to skip the main search bar.

<Preview path="data-filter-pinned" />

### Custom Render Components

You can easily replace standard text boxes with sophisticated components such as a \`Select\` drop-down or a \`DatePicker\` block via the \`render\` prop function. Use \`renderValue\` to specify how the formatted value appears in the badge.

<Preview path="data-filter-custom" />
