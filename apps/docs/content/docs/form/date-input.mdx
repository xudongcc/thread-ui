## Features

- **Label & Description**: Built-in support for labels and helper text
- **Error States**: Display error messages with visual feedback
- **Pre-styled Trigger**: Built-in button trigger with consistent styling
- **Custom Date Formatting**: Flexible date formatting using dayjs format strings
- **Placeholder Support**: Customizable placeholder text when no date is selected
- **Disabled State**: Support for disabling the date input
- **Calendar Integration**: Full-featured calendar with dropdown navigation
- **Keyboard Navigation**: Complete keyboard support for date selection
- **Date Constraints**: Support for `fromDate`, `toDate`, and date range restrictions
- **Controlled & Uncontrolled**: Works in both controlled and uncontrolled modes

## Best Practices

- Always provide a `label` for accessibility
- Use `description` to provide helpful context about the date selection
- Use `error` to display validation errors
- Use `selected` and `onSelect` for controlled state management
- Customize date format using the `format` prop with dayjs format tokens
- Provide clear `placeholder` text to guide users
- Use `fromDate` and `toDate` to limit selectable date ranges
- Use `disabled` prop to prevent interaction when needed
- Customize width using `className` prop if needed

### Calendar Props

The DateInput passes through all Calendar component props. Common props include:

- `fromDate`: Earliest date that can be selected
- `toDate`: Latest date that can be selected
- `disabled`: Dates or date ranges to disable
- `numberOfMonths`: Number of months to display
