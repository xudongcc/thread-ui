## Installation

```bash
pnpm dlx shadcn@latest add @thread-ui/file-upload
```

## Usage

```tsx
import {
  FileUpload,
  FileUploadDropzone,
  FileUploadDropzoneDescription,
  FileUploadDropzoneIcon,
  FileUploadItem,
  FileUploadList,
  FileUploadPreview,
  FileUploadPreviewItem,
} from "@/components/thread-ui/file-upload";
```

```tsx
<FileUpload multiple accept="image/*,.pdf">
  <FileUploadDropzone>
    <FileUploadDropzoneIcon />
    <FileUploadDropzoneDescription />
  </FileUploadDropzone>
  <FileUploadList>
    <FileUploadItem />
  </FileUploadList>
</FileUpload>
```

## Examples

### Multiple

Use the `multiple` prop to allow several files. New dropped, pasted, or selected files are appended to the current list.

<Preview path="file-upload-multiple" />

### Preview

Use `FileUploadPreview` to render image and video previews in a grid.

```tsx
<FileUpload multiple accept="image/*,video/*">
  <FileUploadDropzone />
  <FileUploadPreview>
    <FileUploadPreviewItem />
  </FileUploadPreview>
</FileUpload>
```

<Preview path="file-upload-preview" />

### Disabled

Use the `disabled` prop to disable click, drag-and-drop, and paste handling.

## Features

- **Click to Browse**: Opens the native file picker from a focused drop zone
- **Drag & Drop**: Accepts dropped files and appends them when `multiple` is enabled
- **Paste Support**: Reads file items from the clipboard when focused
- **Controlled & Uncontrolled**: Supports `value`, `defaultValue`, and `onChange`
- **Native Input Props**: Supports `accept`, `multiple`, `disabled`, `name`, and other file input props

## Best Practices

- Use `multiple` when users may provide more than one file
- Use `accept` to guide file selection, but validate files on the server too
- Pair the upload area with a `FieldLabel` and `FieldDescription` for clear form context
- Treat `value` as the source of truth in forms that need external validation
